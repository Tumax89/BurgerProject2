эхлээд orderPage-ийг ачаалах үед тус хуудас load_orders action-ийг dispatch хийнэ, энэ нь order-уудыг серверээс авъя гэдэг захиалга өгч байна гэсэн үг. Тэгээд load_orders middleware(logging) руу орсоны дараагаар middleware(thunk) руу орсоны дараагаар thunk load_orders_start гэдэг тусгай функц цацна, энэ action нь яг бусад action-ны явдаг замаар явна. middleware(thunk) нь яг л orderPage шиг dispatch ашиглан order-ийг цацна, 
thunk load_orders_start гэдэг тусгай функц цацна гэдэг нь энийг нийт системд мэдэгдэнэ гэсэн үг, үүний дараагаар өөрөө сервер лүү хүсэлт явуулна. Хүсэлт явуулаад хүлээж байх хооронд load_orders_start action эхний middleware-ийг өнгөрнө, дараагаар thunk дээр өөр дээр нь хүрч ирэх үед thunk тоохгүй бөгөөд тусгай цункц биш энгийн функц байна гэж үзэн цаашаа явуулна(зарим тохиолдолд дахин шалгадаг код бичиж болно), load_orders_start үүний дараагаар Reducer дээр ирнэ, Reducer энэ үед firebase-ээс өгөгдөл татаж байгаан байна гэж үзээд order Reducer дотор 2 ширхэг state хадгалсан байгаа  учраас load_orders_start эхэлсэн гэж үзээд loading гэдэг state-ийг true болгоно, үүний дараагаар orderPage loading-ийг true болгосныг мэдээд loading true болсон юм чинь spinner-ийг гаргау ирнэ, Spinner гарсааас хойш хэсэг хугацааны дараагаар сэрверээс өгөгдөл ирнэ, амжилттай ирсэн гэж үзээд бүх order-ууд JSON хэлбэрээр ирнэ, thunk энэ үед axios  дотроо хүлээж авсаны дараагаар диспачаараа дахиад load_order_succes гэдэг action үүсгэнэ, энэ айкшин нь энгийн айкшин бөгөөд дотроо серверээс ирсэн payload буюу өгөгдөлийг хадгалсан байна. энэ айкшин нь middleware-үүдээр дамжин redcuer дээр ирнэ,  reducer рүү орсоны дараагаар reducer серверээс ирсэн orders massive-ийг харж байгаад стэйтээ өөрчлөх ба энэ мэдээллийг ordersPage mapStateToprops-оороо дамжуулан мэдэж авах ба spinner-ээ байхгүй болгоно, loading-оо false болгож байна гэсэн үг ба энэ үетэй зэрэгцэн стэйт өөрчлөгдсөн учраас тэр өөрчлөгдсөн order-уудийг дэлгэцэн дээр гаргана.
Хэрвээ өгөгдөл сервэрээс алдаатай ирсэн тохиолдолд thunk алдааны мэдээллийг бас л action байдлаар системд мэдээлнэ, тэгээд reducer-ийг өнгөрсөний дараагаар orderPage дээр очих үед spinner уншихаа байж алдааны мэдээлэл гарч ирнэ.
